# Plan: Capture and Display StructuredOutput Tool

## Summary

Add proper handling for the `StructuredOutput` tool in the log parsing and display system. This is a special tool used by Claude Code when `--output-format json --json-schema` is specified. Currently it appears as an "unidentified tool" with a generic gray help circle icon. We'll add a dedicated `StructuredOutput` tool type with contextual icons and display text based on what the agent is outputting.

## Files to Modify

1. **`crates/orkestra-core/src/workflow/domain/log_entry.rs`** - Add new `StructuredOutput` variant to `ToolInput` enum
2. **`crates/orkestra-core/src/workflow/services/session_logs.rs`** - Parse `StructuredOutput` tool in `parse_tool_input()`
3. **`src/types/workflow.ts`** - Add TypeScript type for `StructuredOutput` tool input
4. **`src/components/LogEntryView.tsx`** - Add icon, color, and display text for `StructuredOutput` with type-specific icons

## Implementation Steps

### Step 1: Add Rust Domain Types
In `crates/orkestra-core/src/workflow/domain/log_entry.rs`:

Add new variant to `ToolInput` enum:
```rust
/// Structured output generation (final agent response).
StructuredOutput {
    /// The output type (e.g., "plan", "summary", "questions", "subtasks")
    output_type: String,
},
```

### Step 2: Parse StructuredOutput Tool
In `crates/orkestra-core/src/workflow/services/session_logs.rs`:

Add handling in `parse_tool_input()` function:
```rust
"StructuredOutput" => {
    // Extract the "type" field from the input to know what kind of output
    let output_type = input
        .get("type")
        .and_then(|t| t.as_str())
        .unwrap_or("unknown")
        .to_string();
    ToolInput::StructuredOutput { output_type }
}
```

### Step 3: Add TypeScript Types
In `src/types/workflow.ts`:

Update `ToolInput` union type:
```typescript

```

2. Create a new `StructuredOutputDisplay` component with type-specific icons and colors:

```typescript
/**
 * Get icon and styling for structured output based on output type.
 */
function getStructuredOutputStyle(outputType: string): {
  icon: React.ReactNode;
  color: string;
  textColor: string;
  label: string;
} {
  const iconProps = { size: 14, strokeWidth: 2.5 };

  switch (outputType.toLowerCase()) {
    // Artifacts - purple/violet theme
    case "plan":
      return {
        icon: <FileOutput {...iconProps} />,
        color: "bg-violet-600",
        textColor: "text-violet-300",
        label: "Generating plan",
      };
    case "summary":
      return {
        icon: <FileOutput {...iconProps} />,
        color: "bg-violet-600",
        textColor: "text-violet-300",
        label: "Generating summary",
      };
    case "verdict":
      return {
        icon: <FileOutput {...iconProps} />,
        color: "bg-violet-600",
        textColor: "text-violet-300",
        label: "Generating verdict",
      };

    // Questions - yellow/amber theme
    case "questions":
      return {
        icon: <MessageCircle {...iconProps} />,
        color: "bg-amber-600",
        textColor: "text-amber-300",
        label: "Asking questions",
      };

    // Subtasks/breakdown - teal theme
    case "subtasks":
      return {
        icon: <Network {...iconProps} />,
        color: "bg-teal-600",
        textColor: "text-teal-300",
        label: "Presenting task breakdown",
      };
    case "breakdown":
      return {
        icon: <Network {...iconProps} />,
        color: "bg-teal-600",
        textColor: "text-teal-300",
        label: "Presenting task breakdown",
      };

    // Terminal states - red theme
    case "failed":
      return {
        icon: <XCircle {...iconProps} />,
        color: "bg-red-600",
        textColor: "text-red-300",
        label: "Task failed",
      };
    case "blocked":
      return {
        icon: <AlertCircle {...iconProps} />,
        color: "bg-orange-600",
        textColor: "text-orange-300",
        label: "Task blocked",
      };

    // Control flow - blue theme
    case "restage":
      return {
        icon: <RotateCcw {...iconProps} />,
        color: "bg-blue-600",
        textColor: "text-blue-300",
        label: "Requesting restage",
      };

    // Unknown/generic - gray theme
    default:
      return {
        icon: <Send {...iconProps} />,
        color: "bg-indigo-600",
        textColor: "text-indigo-300",
        label: `Generating ${outputType}`,
      };
  }
}
```

3. Update `getToolIcon()` and `getToolColor()` to handle `structuredoutput`:

4. Update `ToolInputSummary` to handle the new type:
```typescript

  return <span className={`${style.textColor} text-xs`}>{style.label}</span>;
}
```

5. Special handling in the `tool_use` case to use the type-specific icon and color:
```typescript
case "tool_use": {
  // Special handling for StructuredOutput to get type-specific styling
  let icon = getToolIcon(entry.tool, 14);
  let color = getToolColor(entry.tool);

  if (entry.input.tool === "structured_output") {
    const style = getStructuredOutputStyle(entry.input.output_type);
    icon = style.icon;
    color = style.color;
  }

  return (
    <div className="py-1.5 flex items-start gap-2">
      <span className={`flex-shrink-0 w-5 h-5 rounded flex items-center justify-center text-white ${color}`}>
        {icon}
      </span>
      <div className="flex-1 min-w-0">
        <ToolInputSummary input={entry.input} />
      </div>
    </div>
  );
}
```

## Risks/Considerations

1. **Schema field availability**: The `type` field in `StructuredOutput` input comes from the agent's JSON schema response. Need to verify this is consistently present in the tool input (not just the result).

2. **Backwards compatibility**: Old sessions without `StructuredOutput` will continue to show as "Other" which is fine. New sessions will show the improved display.

3. **New lucide-react icons**: Need to verify the icons (`FileOutput`, `Network`, `RotateCcw`) are available in the project's version of lucide-react. Check `package.json` for version if needed.
