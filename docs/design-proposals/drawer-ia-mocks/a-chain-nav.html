OAuth 2.0 Login Implementation

Added complete OAuth 2.0 authentication flow with GitHub as the initial provider. Redirect URIs are now fully configurable via environment variables.

Changes made

  • Redirect URI read from OAUTH_REDIRECT_URI env var (no hardcoding)
  • Token exchange with retry + exponential backoff
  • Graceful degradation when GitHub is unavailable — login page shows provider status
  • Session hydration updated to handle OAuth-issued tokens

How it works

The numbered dots in the header are the navigation. Click dots 1 or 2 to browse past iterations. A banner appears indicating you're in historical view — no actions available.

Click dot 3 (or "← Current") to return to the action state.

Past iteration tabs: Artifact · Feedback · Logs
Current iteration tabs: Artifact · Diff · Logs

Key tradeoff: Works great when there are a small number of iterations. Gets crowded at 5–6+. Stages not visible at all — only the current stage's iterations shown.
