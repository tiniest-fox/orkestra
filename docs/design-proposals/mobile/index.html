Design Proposals

Orkestra Mobile

Three design concepts for an Orkestra mobile experience, produced by a research → UX → visual → creative direction pipeline. Each concept approaches the same core problem differently: how do you make AI agent oversight feel effortless on a phone?

Swipe Focus

Latest

The pager + activity list. A home screen showing live agent activity, with a swipeable full-screen card pager for everything that needs your attention. One card, one decision, swipe to the next. Diff is a drill-down from the plan review card, not another swipe step.

Designed for the flow: check what's running → review a plan → answer a question → approve a diff → merge a PR.

Pulse — The Decision Inbox

Selected

An inbox you clear, not a dashboard you watch. The Action filter is the default — every card is a pending decision. Inline approve/reject and inline question-answering mean you never navigate to take the most common actions. The stripe → badge → button reading sequence encodes urgency, state, and action in one glance.

Emotional register: decisive, surgical, rewarding.

Stack — The Focus Queue

Selected

Each task that needs you gets the whole screen. Full-screen immersive cards. The canvas washes green on approve swipe, red on reject. Physics-based spring curves create muscle memory. Visual asymmetry between Approve (accent fill) and Reject (muted border) encodes the happy path.

Emotional register: calm focus, quiet confidence, deliberate decisiveness.

Flux

Reference

Variable card heights as a communication mechanism. An earlier iteration (Direction D) where physical card height communicates urgency — tall cards demand attention, compact rows fade into the background. Its key ideas were absorbed into Pulse and Stack. Kept as reference.

The all.html cascade is the clearest demonstration of the height-as-hierarchy concept.
