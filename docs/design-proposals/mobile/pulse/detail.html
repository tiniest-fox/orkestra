Token Refresh Refactoring

Refactored the authentication token refresh flow to eliminate the race condition that was causing duplicate refresh requests. The core issue was that multiple concurrent API calls could each independently detect an expired token and attempt a refresh simultaneously.

The solution uses a mutex-guarded promise that allows the first refresh call to proceed while all subsequent callers await the same promise. Once resolved, all callers receive the new token and the mutex is cleared for the next expiry cycle.

src/auth/token_manager.rs
async fn refresh_token(&self) -> Result<Token> {
let mut guard = self.refresh_mutex.lock().await;
if let Some(token) = &*guard {
return Ok(token.clone());
}
let token = self.do_refresh().await?;
*guard = Some(token.clone());
Ok(token)
}
