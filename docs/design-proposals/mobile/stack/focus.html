Fix auth token refresh

handy-fox · hotfix · 3m ago

Refactored the authentication token refresh flow to eliminate the race condition causing duplicate refresh requests when multiple concurrent API calls each independently detect an expired token.

token_manager.rs
async fn refresh_token(&self) {
let mut guard = self.mutex.lock().await;
if let Some(t) = &*guard { return Ok(t.clone()); }
let t = self.do_refresh().await?;
*guard = Some(t.clone()); Ok(t)
}

All 14 existing auth tests pass. Added 3 new concurrent-refresh tests. The race condition is no longer reproducible under load testing.
