Add JWT validation middleware

brave-owl · hotfix · 3m ago

Added JWT validation middleware that intercepts all protected routes. Tokens are verified against RS256 signatures using cached JWKS keys fetched from the identity provider's discovery endpoint.

The middleware extracts the Bearer token from the Authorization header, validates its signature, checks expiry and issuer claims, and injects the decoded claims into the request context for downstream handlers.

Changes

  1. Created JwtValidator struct with a JWKS cache that refreshes keys every 6 hours or on key-not-found. Uses jsonwebtoken crate for RS256 verification.
  2. Implemented validate_request() middleware that returns 401 with WWW-Authenticate header on failure, including the specific error reason (expired, invalid signature, missing claim).
  3. Added claims extraction into RequestExt::claims() so route handlers can access user ID, roles, and scopes without re-parsing.
middleware/jwt.rs
pub async fn validate_request(
req: Request,
validator: &JwtValidator,
) -> Result<Request, AuthError> {
let token = req
.header(AUTHORIZATION)
.and_then(|h| h.strip_prefix("Bearer "))
.ok_or(AuthError::MissingToken)?;
let claims = validator
.validate(token).await?;
Ok(req.with_claims(claims))
}
