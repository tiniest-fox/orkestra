Database Schema Update Plan

Overview

Add session tracking and structured logging tables to support agent process recovery and debugging. This is a non-breaking, additive change -- no existing tables are modified.

Changes

  1. Add workflow_stage_sessions table for agent session tracking
  2. Add log_entries table for structured agent logs with typed entries
  3. Add composite indexes on (task_id, stage) for both tables
  4. Add foreign key constraints referencing workflow_tasks(id)

Migration Strategy

Create a single migration file following Refinery conventions:

V15__add_sessions_and_logs.sql
  • Run migration on startup (Refinery handles this automatically)
  • No data migration needed -- these are new tables only
  • Rollback: drop both tables (no data loss since they're new)
