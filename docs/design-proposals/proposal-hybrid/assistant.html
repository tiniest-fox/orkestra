You
Can you help me understand how the subtask system works? Specifically how parent tasks know when all their children are done.
Assistant

The subtask lifecycle works in two phases: creation and completion detection.

When a stage with subtasks capabilities approves output, SubtaskService creates child tasks with dependencies encoded between them. Each child gets its own worktree, branched from the parent's branch.

Completion detection runs on every orchestrator tick via advance_completed_parents(). It checks if all subtasks for a parent have status Done — if so, it advances the parent to the completion_stage configured in the workflow.
