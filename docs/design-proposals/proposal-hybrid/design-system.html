Forge Design System
Orkestra UI — Canonical Reference

This is the definitive design system reference for Forge — the Orkestra desktop interface. Every token, component, and state documented here is authoritative. All screens must be built from these primitives. When in doubt, come back here. Do not invent new tokens. Do not introduce new colors. Do not add radius values not in this system.

03

Status Symbols

Seven glyphs. No icons. No icon font dependency. Each glyph is unambiguous at any size, loads instantly, and can be read in peripheral vision. The signal color carries the semantic meaning — the glyph carries the type distinction.

*
Working
Agent actively running. Pulses in amber at 2.5s cycle. Matches active pipeline segment color.
?
Questions
Agent has asked questions. Blocked until user responds.
!
Failed
Agent or script failed. User attention required.
>
Review
Artifact ready. Waiting for approve or reject.
.
Done
Pipeline complete. Task integrated and closed.
~
Idle
Queued, waiting on dependencies, or not yet started.
-
Archived
Task archived. Does not appear in active feed.
Animation spec

Only the * working symbol animates. CSS animation: pulse 2.5s ease-in-out infinite. Opacity oscillates 0.4 to 1.0. All other symbols are static. No other decorative motion in the list view.

Spec

Full pipeline: 6px height, segments separated by 3px gap, each segment border-radius: 3px. Compact pipeline in task rows: 4px height, 20px min-width per segment, 2px gap, border-radius: 2px. Active segment animation: opacity 0.55–1.0, 1.5s ease-in-out infinite. No border. No shadow. No tooltip needed — label row provides stage identity.

05

Buttons

Every button that has a keyboard equivalent shows its keybinding. This is non-negotiable. Buttons have 6px radius. Never 0px (too harsh), never 12px (too soft for a power tool). Hover transitions are 80ms — fast enough to feel responsive, not so fast they feel glitchy.

06

Task Row

The core list item. Four columns: symbol, title+meta, pipeline bar, badge. Minimum height 40px. No cards, no shadows, no border on individual rows. The focused row gets a 2px left border in accent orange — the only row-level color treatment. Completed rows are dimmed to 50% opacity.

Row anatomy

Column 1 (18px): Status symbol in signal color. IBM Plex Mono 14px 600. Column 2 (minmax 0, 200px): Title (IBM Plex Sans 500 13px primary) + ID (IBM Plex Mono 10px muted). Column 3 (148px): Compact pipeline bar — 4px tall segments, flex, 2px gap. Stage label right-aligned in mono 9px. Column 4 (minmax 0, 1fr): Live activity text — IBM Plex Mono 11px muted. Truncates. Column 5 (auto): Actions — pill buttons or status badge. Focused row: border-left: 2px solid var(--accent) plus accent-bg tint. Row padding: 8px vertical, 24px horizontal.

Spacing Scale

4px base unit. All values are multiples of 4. Use named spacing tokens — do not invent arbitrary pixel values. When a value isn't in the scale, use the nearest step and document the exception.

--sp-1
4px — dense gap (pipeline segments, symbol-to-content)
--sp-2
8px — tight gap (icon+label, meta items, section gap)
--sp-3
12px — element gap (button group, form fields)
--sp-4
16px — row padding, list item padding
--sp-5
20px — command bar padding, outer column padding
--sp-6
24px — section margin, right pane padding
--sp-8
32px — major section breaks
--sp-10
40px — document-level padding
--sp-12
48px — page margin, full-width section padding

08

Section Layout

The feed is organized into three intent-based sections. Section headers are 11px IBM Plex Sans 600 uppercase, color text-tertiary. Sections are separated by a 1px border-color divider — not whitespace alone, not a heavy separator. The command bar and status line are the persistent chrome — everything else scrolls.

Feed Mode

Full-width. Command bar 44px fixed. Status line 28px fixed. Feed scrolls. Section headers are sticky within scroll. Max three sections: Needs Attention, Active, Completed Today.

Split Mode (Ctrl+\)

Left pane: 360px fixed. Right pane: 1fr, scrollable. Divider: 1px border-color. Focused row gets inset left border (2px accent). Same section structure in left pane, compressed rows.
