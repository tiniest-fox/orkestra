feat: add JWT refresh token rotation

Implements single-use refresh token rotation as a defense against token theft. When a refresh token is used, it is immediately revoked and a new pair (access + refresh) is issued. Any attempt to reuse the old token returns 401 and invalidates the entire session — a stolen token cannot silently persist after a legitimate refresh.

The previous behavior issued a new access token from a long-lived refresh token that was never rotated, leaving a 7-day window where a stolen token remained valid even after the legitimate client had refreshed.

The rotate_refresh_token() method in TokenStore is atomic: revoke old, issue new pair, return both. handle_refresh in the middleware now returns both tokens in the response so clients can update storage in one round-trip.
