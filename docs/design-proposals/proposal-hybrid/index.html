The Operator

Hybrid Design Proposal for Orkestra
Keyboard-first density + typographic warmth. "Linear meets Warp terminal."
Core Views
> feed.html primary
Main task feed -- intent-grouped list with pipeline visualization and live status
> task-detail.html
Full task detail in focus mode -- working state with live activity and metrics
> review.html
Review mode -- artifact content with approve/reject action bar
> questions.html
Questions mode -- answering agent questions with keyboard selection
State Views
> failed.html
Failed task -- error display with activity log and retry options
> subtasks.html
Parent task with subtask hierarchy, dependency graph, and progress tracking
> monitoring.html
Active agent monitoring with live tool calls and file edits
Utilities
> diff.html
Code diff viewer with file tree navigation and syntax highlighting
> logs.html
Full agent session log with type filters and session navigation
> create.html
Task creation -- command bar quick-create with flow picker
Each file is self-contained HTML/CSS. Open any directly in a browser.
All views include dark/light mode toggle. Design system defined in README.md.
Sources: Proposal A (density, pipeline), B (warmth, typography), C (keyboard, minimalism).
