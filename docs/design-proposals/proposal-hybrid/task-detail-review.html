Rate Limiting Implementation Plan

Overview

Add per-user and per-endpoint rate limiting to the API using a sliding window algorithm stored in Redis. This prevents abuse of resource-intensive endpoints while maintaining a good experience for legitimate clients. Limits are surfaced via standard X-RateLimit-* headers and a 429 Too Many Requests response with a Retry-After header indicating when the client may retry.

Changes

  1. Add rack-attack gem for rate limiting middleware — provides throttle DSL, Redis store support, and customizable response bodies
  2. Configure per-user limits: 1000 req/hour for standard tier, 5000 req/hour for premium tier, keyed on current_user.id
  3. Add per-endpoint overrides for resource-intensive endpoints: /api/reports/generate at 10/hour, /api/exports at 20/hour
  4. Implement custom error response: 429 status with JSON body { "error": "rate_limit_exceeded", "retry_after": N } and Retry-After header set to seconds until window resets
  5. Feature-flag entire rollout behind rate_limiting_enabled — deploy to staging with flag off, enable for 10% of users, then full rollout

Files Modified

config/initializers/rack_attack.rb new — throttle rules, Redis store config, custom responder
app/controllers/application_controller.rb add rate limit headers to all responses
config/features.yml add rate_limiting_enabled flag (default: false)
spec/requests/rate_limiting_spec.rb new — integration tests for throttle behavior and headers
Gemfile add rack-attack ~> 6.7

Rollout

Feature-flagged behind rate_limiting_enabled. Deploy to staging first with the flag disabled. Verify Redis connectivity and throttle rule behavior in staging using a load test script. Enable for internal users, then 10% of production traffic, then full rollout over 72 hours. Monitor rack_attack.throttle metrics in Datadog during each phase.

Rollback

Set rate_limiting_enabled to false in the feature flag service. No database migrations — Redis throttle buckets expire naturally. Rack::Attack middleware remains in the stack but all throttle rules are guarded by the flag check and short-circuit immediately when disabled.
