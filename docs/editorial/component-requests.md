# Component Requests

Validated requests for components that don't exist yet. Maintained by the compound agent.

Each entry captures the original use case and accumulates support from subsequent writers who hit the same gap. When someone creates a Component Trak to build a requested component, the entry moves to **In Progress**. Once built and added to `editorial/components.md`, the entry is removed from this file.

---

## How to use this file

**If you need a component that doesn't exist:**

1. Check this file first. If a request already covers your use case, add your use case to the "Also needed for" list rather than filing a duplicate.
2. If nothing matches, document the gap in your draft artifact under `Component Requests`. The compound agent will add it here after the editor validates it.

**Request format:**

```markdown
## [Component name]

**What it should do:** One-sentence description of the component's purpose.

**Originally needed for:** [Page title] — [what the writer was trying to express and couldn't]
**Workaround used:** [What they did instead, so future writers know what the current fallback is]
**Suggested behavior:** Props, visual description, interaction notes — enough for a developer to spec it.

**Also needed for:**
- [Page title] — [use case that +1s this request]

**Status:** Pending
```
